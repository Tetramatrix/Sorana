What is Sorana?

Sorana is a visual workspace application that uses artificial intelligence for spatial organization of digital files. It features a WYSIWYG canvas editor, no-code agent orchestration, and supports multiple AI backends including OpenAI, Mistral, and Ollama.

Core Capabilities

  • Spatial AI Organization: Uses AI to semantically group files and arrange them on a 2D canvas, providing a visual overview of projects.
  • Visual Canvas Editor: Direct manipulation of workspace elements. Drag and drop files between groups, create new categories, rename items, and adjust group boundaries.
  • Model Management: Connect to multiple cloud and on-prem LLM backends including OpenAI, Mistral, Ollama, and other compatible services.
  • Visual Agent Orchestration: Create custom agents and connect them into processing pipelines using a drag-and-drop interface.
  • Document Interaction: Interact with files (PDFs, code, text) to extract information and insights.
  • Structure Visualization: Generate visual representations of folder hierarchies to understand relationships and organization.
  • On-Prem & Portable: Run AI models on-prem or connect to cloud services. The application is portable and keeps data under user control.

Intended Users

Sorana is designed for users who need to organize and analyze complex digital information:

  • Developers: Organize codebases, document projects, and create automation workflows.
  • Researchers: Manage research papers, datasets, and notes with visual organization.
  • Professionals: Organize project files, documents, and resources with visual tools.
  • Creatives: Organize digital assets and project files in a visual workspace.

Operation

The application functions through these steps:

  1. Connect folders and files to the workspace.
  2. AI Processing organizes content spatially and semantically.
  3. Visualize projects as interactive layouts.
  4. Interact with documents to extract information.
  5. Create automation workflows by connecting components.
  6. Configure workflow settings.

Technical Implementation

AI Model Performance

Sorana uses AI models to group and arrange files and folders. The built-in default model is lightweight and works offline, but may not always classify files perfectly, especially for less common file types. From 8b parameters, reasonable results can be expected. Users can connect larger, on-prem or cloud models for improved accuracy. Processing time varies based on folder size and available hardware.

  • Built-in Models: The built-in model (~806MB) is downloaded on first run and works fully offline. It is fast, but may sometimes classify complex files as "Miscellaneous".
  • AI Integration: Supports connection to both on-prem and cloud-based AI services:
    • Built-in Models: Pre-configured models accessible through the model manager.
    • On-Prem Services: Ollama, Llamacpp, and other self-hosted LLM solutions
    • Cloud Services: OpenAI, Mistral, and other cloud-based AI platforms
    • Configuration: Switch between different AI backends based on requirements

Note: Larger models can provide better grouping, but require more memory and processing power.
