Inspiration

- -

We were frustrated by the limitations of traditional file managers. For decades, we've been forced to organize digital lives into rigid, list-based hierarchies and nested folders that hide information rather than reveal it. Our brains don't work in lists; they work through associations and spatial relationships. We wanted to build a system that reflects this—moving away from "where did I save that file?" to "what is this project about?" by using semantic and visual grouping to reveal the hidden structure of our data.

What is Sorana?

- -

Sorana is an AI-powered visual workspace featuring a built-in MCP server for file operations, web content tools, OCR document processing, spatial AI organization, and no-code agent orchestration. Now with MCP Manager for visual server configuration, Gmail MCP server for full agentic email control, intelligent router & intent engine for smart tool routing, and adaptive 4-tier AI memory that auto-learns from conversations while using 93% fewer tokens.

Technical Implementation

- -

AI Model Performance

Sorana uses AI models to intelligently group files and folders based on semantic meaning and logical relationships. The built-in default model is lightweight and works offline, but may not always classify files perfectly, especially for less common file types. For optimal performance, we strongly recommend using Llama 3.1 8b Instruct or higher models. Processing time depends entirely on the folder size - fewer files mean faster processing times. Naturally, the best performance is achieved with cloud services.

AI Integration

Sorana supports connection to both on-prem and cloud-based AI services:

  • Built-in Models: Pre-configured models accessible through the model manager
  • Local and On-Prem Services: 🍋 Lemonade (with FastFlowLM NPU), Llama.cpp, Ollama, and other self-hosted LLM solutions
  • Cloud Services: OpenAI, Mistral, Google Gemini, and other cloud-based AI platforms

Hardware Requirements

  • Built-in Models: Range from 1B parameter (806MB) to 20B parameter (8-16GB)
  • Recommended 8B Models (e.g., Llama 3.1 8b Instruct): Minimum 8 GB RAM or 4 GB VRAM for smooth operation
  • Hardware Requirements Increase with Model Size: Larger models with more parameters require higher specifications
  • Cloud Models: No local hardware requirements (requires internet connection)

Recommendation

🍋 Lemonade For optimal local and on-prem performance with both GPU and NPU acceleration, we recommend:

🍋 Lemonade ⚡ FastFlowLM

🍋 Lemonade is AMD's new high-performance AI server (C++ based) with integrated backends including Llama.cpp, Whisper, and FastFlowLM for native Ryzen NPU acceleration.

Note: Larger models can provide better grouping, but require more memory and processing power. Processing time is directly proportional to the number of files - fewer files result in faster processing.

Adaptive 4-Tier AI Memory & Smart Routing

- -

Sorana features an adaptive 4-tier AI memory system that automatically learns from every conversation. Combined with an intelligent router and intent engine, the agent always remembers — from your last message to your last session — while using dramatically fewer tokens. Perfect for local and private inference.

The 4 Memory Tiers

  • Current History — Your active conversation, always in context
  • Episodic Summary — Long chats compressed into compact summaries (93% token reduction)
  • Observations — Individual facts, preferences, and skills auto-extracted from conversations
  • Profiles — Observations consolidated into entity summaries (user, company, topic) — 61% fewer tokens

How it works

  • Auto-Extraction: Every message analyzed — memories saved automatically
  • Profile-First Retrieval: Compact profiles injected first, observations as fallback
  • 4-Tier RAG Search: NPU Embeddings → BM25 → Keyword → Full Text — precise, fast, works offline
  • Conversation Compaction: 17,000 → 1,200 tokens while keeping full context
  • Cross-Agent Memory: Share memories between agents, import/export, global memories

Intelligent Router & Intent Engine

  • Intent Classification: Detects memory search, tool calls, document queries, or hybrid intents
  • Entity Extraction: Automatically identifies companies, users, topics from natural language
  • Smart Tool Routing: Routes to correct MCP server (file ops, Gmail, web tools)
  • Loop Prevention: Max 3 tool calls per request — prevents infinite loops
  • Multi-Language: English + German intent patterns supported

Key Benefits

  • Token Efficiency: Smart memory layering reduces token usage by serving only relevant context
  • NPU Acceleration: Uses 🍋 Lemonade backend with integrated FastFlowLM NPU for Windows Ryzen. 🍋 Lemonade is AMD's high-performance AI server (C++ based) integrating multiple backends including FastFlowLM for native NPU acceleration. Falls back to text-based retrieval if no NPU is available.
  • Multi-Language: English + German for memory routing, 25+ languages for tool intents (Gmail, file ops), 15+ languages for document retrieval — all auto-detected

MCP Server Usage Guide

- -

Sorana includes a built-in MCP (Model Context Protocol) server that enables advanced file operations and web content access through AI agents. Connect any third-party external MCP server (Google Drive, GitHub, PostgreSQL, custom tools) for unlimited extensibility. To use these capabilities:

  1. Open MCP Manager to configure and enable servers
  2. Connect external MCP servers if needed (Google Drive, GitHub, databases, etc.)
  3. Create an agent in the workspace
  4. Right-click on the agent title and select "Chat"
  5. Interact directly with files, folders, web content, and external services

MCP Manager

MCP Manager — Visual interface for managing MCP servers. Enable/disable servers, configure settings, and monitor connection health — all without editing config files. Connect any compatible external MCP server for unlimited extensibility.

Model Manager Split View

The Model Manager features a split view when using 🍋 Lemonade as the backend. Lemonade supports multi-model loading, allowing both a chat model and an embedding model to run simultaneously:

  • Upper Pane: Chat/Completion models (for conversation and reasoning)
  • Bottom Pane: Embedding models (for semantic search and RAG) — runs on NPU or GPU

Embedding models power the 4-tier RAG system's Tier 1 (NPU Embeddings). Without an embedding model, the system gracefully falls back: BM25 → Keyword → Full Text. Currently, only Lemonade supports multi-model loading. Other backends (Ollama, LM Studio, etc.) do not support loading an embedding model alongside a chat model — but the 4-tier RAG system ensures full functionality even without embeddings.

  • "Get size and creation date of config.json" (uses get_file_info)

Gmail MCP Server

Gmail MCP Server — Full agentic Gmail control through your AI agent:

  • List emails by label, date, or search query
  • Get individual email content with headers
  • Send new emails directly from chat
  • Label — add, remove, or manage email labels
  • Trash — move emails to trash
  • Archive — download emails to disk in JSON, EML, or MBOX format

Archive Example:

  • "Archive emails from client after 2024-01-01 before 2024-12-31 with label PROJECT_X_2024"

Why this matters: Google only allows MBOX export for entire Gmail via Google Takeout. With Sorana's MCP server, the agent archives specific emails on demand — perfect for backups, compliance, or knowledge management.

Configuration: Easy setup with JSON config or OAuth2 token. First run triggers authorization, token is cached afterwards.

Document OCR and Context Enhancement

- -

Document OCR and Context Enhancement allows users to enrich agents with specific knowledge by dragging and dropping documents directly onto agents. After organizing files spatially, users can make documents accessible to agents by processing them through OCR, converting them into contextual information. This is accomplished via the context menu options 'Document Overview' and 'Process Documents,' which is particularly valuable for PDFs and image-based documents.

It's important to distinguish this from MCP server functionality. While the MCP server allows file interaction during chats, it currently lacks OCR capabilities and can only access text-based file content.

Document OCR Capabilities

Sorana includes powerful Optical Character Recognition (OCR) capabilities for processing various document types with support for common character encodings:

  • Text PDFs: Extract text from PDF documents (supports embedded text and OCR for scanned content)
  • Plain Text Files: Process .txt files with support for:
    • UTF-8 (recommended for full Unicode support)
    • Latin-1 (ISO-8859-1) as fallback encoding
  • Code Files: OCR support for source code files including:
    • Python (.py), C++ (.cpp), JavaScript (.js), Java (.java)
    • C# (.cs), PHP (.php), Ruby (.rb), Go (.go)
    • TypeScript (.ts), Swift (.swift), Kotlin (.kt)
    • And other common programming language files in UTF-8 or Latin-1 encoding
  • PDFs with Images: Built-in method for OCR processing of PDFs containing images

Requirements for PDF Image OCR

To enable OCR for PDFs with images, you need to install the official Tesseract OCR engine with default settings and ensure it's available in your system PATH. Tesseract is an open-source OCR engine that provides high-quality text extraction from images.

Download Tesseract from:

Encoding Support Notes

The application primarily uses UTF-8 encoding for document processing and falls back to Latin-1 (ISO-8859-1) when UTF-8 decoding fails. For optimal results, we recommend using UTF-8 encoding for your documents. This ensures the best compatibility with international characters and special symbols.
